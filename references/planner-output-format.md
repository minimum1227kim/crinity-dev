# Code Planner 출력 형식

> code-planner 에이전트가 Step 8에서 Read하여 이 형식으로 분석 결과와 개발 계획을 출력한다.

---

```
## 분석 결과

### 영향 모듈
- 백엔드: {architecture.md에서 확인한 실제 모듈 경로}
- 프론트엔드: {frontend-rules.md에서 확인한 실제 소스 경로}

### 기존 유사 구현
- [재사용 가능] {파일경로}: {메서드명} — {설명}
- [참고] {파일경로}: {설명}

### 수정 필요 파일
| 파일 | 작업 | 우선순위 |
|------|------|--------|
| {파일경로} | 신규 생성 / 수정 | 높음/중간/낮음 |

### 리스크
- {리스크 항목}: {대응 방법}

---

## 개발 계획

### 변경 규모
- 수정 파일 수: N개
- 스택: backend-only / frontend-only / fullstack
- 레이아웃 변경: N개

### 파일별 변경 계획
| 파일 | 작업 | 레이어 |
|------|------|------|
| {파일경로} | 신규/수정 | Entity/Service/... |

### 설계 결정 사항
- {결정 사항}: {이유}

### 리스크 및 주의사항
- {리스크}: {대응}

### 테스트 시나리오 (프론트엔드 변경 포함 시)

| # | 시나리오 | 전제조건 | 기대 결과 |
|---|---------|---------|----------|
| 1 | {시나리오명} | {전제조건} | {기대 결과} |

#### 시나리오 1: {시나리오명}
1. {CLAUDE.md에서 확인한 개발 서버 URL} 접속
2. {모듈} 진입
3. {조작1}
4. {조작2}
→ 기대: {구체적 검증 조건}
→ 검증: {텍스트 확인 / 요소 존재 확인 / API 응답 확인}
```
