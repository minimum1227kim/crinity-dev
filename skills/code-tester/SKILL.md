# code-tester Skill

트리거 키워드: "브라우저 테스트", "UI 테스트", "화면 테스트", "동작 확인해줘", "E2E 테스트", "browser test"

## 실행 흐름

테스트 루프 (최대 2회):

1. 테스트 시나리오 소스 결정
   - 우선순위: 사용자 제시 > code-planner 출력 > git diff 기반 자동 생성
   - 자동 생성 시 사용자 컨펌 필요
2. 전제조건 확인 (개발 서버 URL: `.claude/rules/build-commands.md` 참조, 로그인 상태, 테스트 데이터)
3. `browser-tester` 에이전트 실행

## 결과 처리

| 결과 | 처리 |
|------|------|
| 전체 PASS (SKIP 포함) | 완료 (SKIP 항목은 수동 확인 권고) |
| FAIL 1건 이상 | 개발 루프 복귀 (`code-developer` 스킬) |
| 테스트 루프 2회 초과 | 사용자 보고, 수동 테스트 전환 권고 |
