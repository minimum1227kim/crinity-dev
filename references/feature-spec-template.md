## 4. DB 설계서

### 4.1 ERD

```
┌──────────────────┐       ┌──────────────────┐
│  [테이블A]        │       │  [테이블B]        │
├──────────────────┤       ├──────────────────┤
│ PK  id           │───┐   │ PK  id           │
│     [컬럼1]      │   │   │ FK  [테이블A_id]  │──→ 테이블A.id
│     [컬럼2]      │   │   │     [컬럼1]      │
│     [timestamps] │   └──→│     [timestamps] │
└──────────────────┘       └──────────────────┘
```

### 4.2 테이블 정의

#### TB-001: [테이블명] (`[스키마].[물리명]`)

| 항목 | 내용 |
|------|------|
| 테이블명 | [물리명] |
| 설명 | [테이블 용도] |
| 비고 | [파티션, 샤딩 등 특이사항] |

**컬럼 정의**

| No | 컬럼명 | 논리명 | 타입 | 길이 | PK | FK | NN | 기본값 | 설명 |
|----|--------|--------|------|------|----|----|----|--------|------|
| 1 | id | 고유ID | BIGINT | - | Y | - | Y | AUTO_INCREMENT | |
| 2 | [컬럼명] | [논리명] | [타입] | [길이] | - | - | [Y/N] | [기본값] | [설명] |

> 프로젝트의 Entity 컨벤션(ID 전략, 타임스탬프 형식, 멀티테넌시 필드 등)은 rules 파일(backend-rules.md)의 Entity 섹션을 참조하여 작성한다.

**인덱스**

| 인덱스명 | 컬럼 | 유형 | 용도 |
|----------|------|------|------|
| idx_[table]_[name] | [컬럼] | [NORMAL/COMPOSITE/UNIQUE] | [용도] |

**제약 조건**

| 제약조건명 | 타입 | 컬럼 | 참조 | 설명 |
|-----------|------|------|------|------|
| pk_[table]_id | PRIMARY KEY | id | - | |
| fk_[table]_[ref] | FOREIGN KEY | [컬럼] | [참조테이블].[컬럼] | [CASCADE 정책] |
