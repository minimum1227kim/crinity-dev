# code-planner Skill

트리거 키워드: "개발해줘", "만들어줘", "추가해줘", "구현해줘", "기능 개발", "계획해줘", "분석해줘", "설계해줘", "기획해줘", "어떻게 만들까"

## 실행 흐름

### Step 0: 의도 구체화 (에이전트 실행 전, 스킬이 직접 수행)

**스킵 조건** — 아래 중 하나라도 해당하면 바로 Step 1로 진행:
- 사용자가 구체적인 **파일명/메서드명**을 지정한 경우
- 사용자가 **상세한 명세**를 함께 제공한 경우
- **기존 기능의 단순 수정** (버그 수정, 텍스트 변경, 설정값 변경 등)
- 사용자가 **"바로 해줘"**, **"빠르게"** 등 빠른 진행을 명시한 경우

**구체화 진행 시** (최대 3라운드, 라운드당 2~3개 질문):

| 라운드 | 질문 카테고리 | 예시 |
|--------|-------------|------|
| 1 | 범위, 핵심 동작, UI 방향 | "관리자 화면도 필요한가요?", "목록 형태 vs 카드 형태?" |
| 2 | 제약, 예외, 권한 | "기존 데이터 호환성?", "데이터 없을 때 처리?" |
| 3 | 성능, 알림, 연동 (필요 시만) | "실시간 알림이 필요한가요?" |

**구체화 완료 기준** (모두 충족 시):
- 대상 모듈이 특정됨
- 핵심 동작(CRUD, 표시, 설정 등)이 명확함
- UI 배치 방향이 대략 잡힘
- 주요 예외 케이스가 파악됨

구체화 결과를 **"정제된 요구사항"**으로 정리하여 code-planner 에이전트에 전달한다.

---

### Step 1: `code-planner` 에이전트 실행

정제된 요구사항(또는 원본 요청)을 전달하여 코드베이스 탐색 + 아키텍처 설계 + 테스트 시나리오를 수행한다.

---

### Step 2: UI 결정 필요 시 `ui-brainstorm` 스킬 실행

code-planner 에이전트 출력의 **"설계 결정 사항"** 섹션에 UI 옵션이 2개 이상 제시된 경우:
1. `ui-brainstorm` 스킬 실행
2. 사용자 선택 후 결과를 계획의 "설계 결정 사항" 섹션에 반영

---

### Step 3: 사용자 컨펌 대기

⏸ 계획 전체를 제시하고 사용자 컨펌을 기다린다. 수정 요청 시 에이전트 재실행 또는 계획 수정.

---

### Step 4: 컨펌 완료 후 처리

code-planner 에이전트 출력의 **"변경 규모"** 섹션 기준으로 복잡도 threshold 확인:

**충족 시** (수정 파일 5개↑ / 레이아웃 변경 3개↑ / fullstack):
→ `task-manager` 스킬 (create 오퍼레이션) 실행 → `.claude/tasks/{feature-slug}_task.md` 생성

태스크 분해 시 에이전트 출력의 **"파일별 변경 계획"** 레이어 컬럼을 기준으로 아래 규칙을 적용한다:

**에이전트 배정 규칙**

| 레이어 | 배정 에이전트 |
|--------|-------------|
| 백엔드 레이어 (code-planner 에이전트 출력의 "파일별 변경 계획"에서 레이어 확인) | `backend-developer` |
| 프론트엔드 레이어 (code-planner 에이전트 출력에서 레이어 확인) | `frontend-developer` |

**병렬 처리 규칙**

| 스택 유형 | 처리 방식 |
|---------|---------|
| `backend-only` | 순차: code-planner 에이전트가 결정한 레이어 순서 |
| `frontend-only` | 순차: code-planner 에이전트가 결정한 레이어 순서 |
| `fullstack` | 백엔드와 프론트엔드는 **병렬** 가능. 단, 프론트엔드 API 계층은 백엔드 API 계층 완료 후 시작 (API 계약 확인 필요) |

**미충족 시**: 생략
